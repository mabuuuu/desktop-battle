# Desktop Battle - 阵营分裂机制设计文档

## 概述

当一方阵营被全灭后，胜利方成为场上唯一势力。胜利方继续发展，当人口超过阈值时，内部矛盾逐渐积累，最终导致阵营分裂，形成新的敌对势力，开启新一轮战斗。

## 触发条件

### 前置条件

1. 场上仅剩一个阵营（其他阵营全部被灭）
2. 该阵营存活单位数量超过 **人口阈值**

### 人口阈值

| 参数 | 值 | 说明 |
|------|-----|------|
| 分裂触发人口 | 15 | 存活单位数 ≥ 15 时开始积累矛盾 |
| 矛盾积累速率 | 每5秒 +1 | 人口超阈值后自动积累 |
| 人口加成 | (当前人口 - 阈值) × 0.5 | 每多1人额外加速矛盾积累 |
| 矛盾上限 | 100 | 矛盾值上限 |
| 分裂触发矛盾值 | 80 | 矛盾值 ≥ 80 时触发分裂 |

## 矛盾积累机制

### 矛盾值 (Conflict Score)

```python
conflict_score: float  # 0.0 ~ 100.0
```

### 积累公式

```
每5秒:
  if 场上仅剩1方 and 存活单位 >= 分裂触发人口:
    积累 = 1.0 + (存活单位 - 分裂触发人口) * 0.5
    conflict_score += 积累
```

### 矛盾可视化

| 矛盾值范围 | 视觉效果 | 行为变化 |
|-----------|---------|---------|
| 0~30 | 无变化 | 正常运作 |
| 30~60 | 偶尔出现争吵气泡（两个单位间闪红色感叹号） | 1-2个单位偶尔停下争吵（1秒） |
| 60~80 | 频繁争吵，小冲突闪光 | 3-5个单位停下争吵，偶尔互相推搡（小击退） |
| 80+ | 红色警告闪烁，单位间明显敌意 | 触发分裂 |

### 争吵行为

当矛盾值 > 30 时，部分单位会进入"争吵"状态：

```
争吵触发:
  概率: 每秒每单位 conflict_score / 500
  持续: 1~2秒
  表现: 两个单位面对面，头顶红色感叹号
  效果: 暂停当前行为，争吵结束后恢复
```

### 小冲突

当矛盾值 > 60 时，争吵升级为小冲突：

```
小冲突触发:
  概率: 每秒每单位 (conflict_score - 60) / 200
  持续: 0.5秒
  表现: 单位间出现碰撞闪光
  效果: 互相施加小击退（knockback=20），无实际伤害
```

## 分裂机制

### 分裂触发

当矛盾值 ≥ 80 时，下一次更新触发分裂：

### 分裂过程

```
1. 计算分裂人数: 随机 1/4 ~ 1/2 的存活单位
2. 选择分裂方向: 从现有阵营中分离
3. 创建叛军阵营
4. 分裂单位转移阵营
5. 资源点争夺
6. 分裂完成 → 新一轮战斗
```

### 叛军阵营

| 属性 | 值 | 说明 |
|------|-----|------|
| 阵营名 | "{原阵营名}·叛军" | 如"红方·叛军" |
| 主色 | 原阵营主色的互补色 | 红→青, 蓝→橙 |
| 辅色 | 原阵营辅色的互补色 | |
| 初始资源 | 木材10 + 矿石5 | 从原阵营仓库分出 |
| 矛盾值 | 0 | 叛军初始无内部矛盾 |

### 阵营颜色对照

| 原阵营 | 原主色 | 叛军主色 | 叛军辅色 |
|--------|--------|---------|---------|
| 红方 | #FF4444 | #44FFEE (青) | #88FFCC |
| 蓝方 | #4488FF | #FFAA44 (橙) | #FFCC88 |

### 分裂人数

```python
import random
split_ratio = random.uniform(0.25, 0.50)
split_count = max(2, int(alive_units * split_ratio))
```

分裂单位的选择规则：
- 优先选择离原阵营中心最远的单位（边缘单位更容易叛变）
- 已装备武器的单位叛变时带走武器

### 资源点争夺

分裂发生时，叛军尝试抢占一个资源采集点：

```
1. 选择原阵营的一个资源点（优先离叛军单位最近的）
2. 争夺概率: 60%
3. 争夺成功:
   - 资源点转移给叛军
   - 原阵营在该资源点附近的单位被击退
   - 叛军获得该资源点
4. 争夺失败:
   - 叛军不获得资源点
   - 原阵营保留全部资源点
   - 叛军必须在原阵营区域边缘重新建立基地
```

### 分裂后的局面

```
分裂前:
  [红方] ████████████████████  20人, 2资源点

分裂后:
  [红方] ████████████         12人, 1资源点
  [红方·叛军] ███████         8人, 1资源点 (青色)
  
  → 双方立刻变为敌对，开始战斗
```

## 分裂冷却

| 参数 | 值 | 说明 |
|------|-----|------|
| 分裂冷却时间 | 120秒 | 分裂后双方120秒内矛盾值归零，不积累 |
| 二次分裂 | 允许 | 如果叛军又成为唯一势力且人口超阈值，可再次分裂 |

## 游戏循环

```
开局: 红方 vs 蓝方
  ↓ 战斗
红方全灭蓝方 → 蓝方被消灭
  ↓ 胜利方继续发展
红方人口 > 15 → 矛盾积累
  ↓ 矛盾值 ≥ 80
红方分裂 → 红方 vs 红方·叛军(青色)
  ↓ 新一轮战斗
一方全灭 → 胜利方继续发展 → 可能再次分裂...
```
