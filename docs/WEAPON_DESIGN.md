# Desktop Battle - 武器设计文档

## 设计原则

1. **制作链驱动**：所有武器必须通过制作链获得，不能凭空产生
2. **观赏性优先**：低伤害 + 高血量，单次战斗持续30秒以上
3. **差异化**：不同武器有明显的攻击方式、范围、动画差异
4. **成本与效果挂钩**：更强的武器需要更多资源和更高级的工具台

## 武器属性模型

```python
class WeaponSpec:
    name: str               # 武器名称
    damage: float           # 单次伤害
    attack_range: float     # 攻击距离 (px)
    attack_speed: float     # 攻击间隔 (秒)
    knockback: float        # 击退力
    crafting_cost: dict     # 制作消耗 {资源类型: 数量}
    crafting_station: str   # 所需制作台
    crafting_time: float    # 制作时间 (秒)
    visual_length: int      # 视觉长度 (px)
```

## 伤害与观赏性平衡

### DPS 计算表

| 武器 | 伤害 | 间隔 | DPS | 对1000HP击杀时间 |
|------|------|------|-----|-----------------|
| 徒手 | 2.5 | 1.5s | 1.67 | 600s (10分钟) |
| 长矛 | 4 | 2.0s | 2.0 | 500s (8.3分钟) |
| 剑 | 6 | 1.2s | 5.0 | 200s (3.3分钟) |
| 盾+徒手 | 1.5 | 1.5s | 1.0 | 1111s (18.5分钟) |
| 剑 vs 盾 | 3.6 | 1.2s | 3.0 | 333s (5.5分钟) |

**核心设计**：1v1 徒手战斗需要10分钟才能分出胜负，确保长时间观赏性。实际战斗中通常是多对多，且有武器差异，战斗时间在1-5分钟。

### 伤害随机性

每次攻击伤害在 `base_damage * 0.8 ~ base_damage * 1.2` 范围内随机浮动，增加战斗的不确定性和观赏性。

## 制作链

```
资源采集点
    ↓ 采集 (木材/矿石)
    ↓
┌─────────────────────────────────────┐
│  工具台 Lv1 (木材 15)               │
│  ├─ 长矛 (木材 10)                  │
│  └─ 盾 (矿石 10)                    │
├─────────────────────────────────────┤
│  工具台 Lv2 (木材 15 + 矿石 10)     │
│  ├─ 剑 (木材 5 + 矿石 8)            │
│  └─ 弓 (木材 12 + 矿石 3) [V2+]     │
├─────────────────────────────────────┤
│  工具台 Lv3 (矿石 20 + 木材 10)     │
│  └─ 巨锤 (矿石 15) [V2+]            │
└─────────────────────────────────────┘
```

### 制作流程

1. 单位携带足够资源 → 走到工具台旁
2. 选择要制作的武器
3. 扣除资源，开始制作计时
4. 制作完成 → 武器出现在工具台旁
5. 空手单位自动拾取武器

## 武器掉落

- 单位死亡时，武器掉落在死亡位置
- 空手友方/敌方单位可以拾取
- 武器在地面上30秒后消失
